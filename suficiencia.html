Programación IV: Examen de suficiencia

Durante el curso de programación IV se introduce al alumno al desarrollo de aplicaciones web y se espera que:

  • Dado un problema, logre representar los datos involucrados utilizando un modelo formal (relacional o key-value).
  • Pueda utilizar un framework web para construir una aplicación que interactúe con esos datos de manera eficiente y modular: separando datos de presentación y lógica de negocios.
  • Sepa aplicar el modelo de arquitectura REST para la manipulación de los recursos
  • Conozca el protocolo HTTP y los riesgos (y medidas preventivas) de seguridad aplicables al desarrollo web.
  • Pueda estructurar documentos semánticamente correctos en html.
  • Pueda aplicar estilo de forma eficaz mediante hojas de estilo css.
  • Pueda agregar interacción del lado del cliente mediante javascript.
  • Pueda utilizar un sistema de control de versiones colaborativo.
  • Sea capaz de publicar una aplicación

Este examen de suficiencia pretende identificar estas competencias.

Política de entrega

El alumno tiene 24 horas para presentar el examen al docente y deberá poner el código fuente final en github.com.

Descripción del examen

Reproducir un sub-conjunto de la funcionalidad del sitio web writeboard.com, con las características enumeradas a continuación (el puntaje se muestra en cursivas al final de cada elemento) :

  1. Formulario de creación de una writeboard: se deberá asignar una url única a la writeboard, así como un nombre y una contraseña. 20
  2. Edición de una versión inicial writeboard utilizando algún lenguaje de marcado (p.e. textile, markdown o restructured text) y posibilidad de guardarla, asociándola a un autor.20
  3. Posibilidad de editar una writeboard, creando una nueva versión cada vez, con posiblemente otro autor.10
  4. Listar las versiones de una writeboard creadas hasta la fecha, permitiéndole al usuario verlas.10
  5. Comparar dos versiones de una writeboard, utilizando algún algoritmo o librería de diff y mostrar los cambios entre éstas. 10
  6. Permitir al usuario ver, con javascript, algún diálogo de ayuda sobre el lenguaje de marcado elegido.5

En cuanto a la implementación concreta de la solución, se evaluará que:

  • Los documentos html sean semánticamente correctos y el estilo se aplique exclusivamente con css.10
  • Las urls reflejen los recursos involucrados y las acciones concretas, siguiendo el modelo de arquitectura REST.5
  • El modelado de datos respete o el esquema relacional de bases de datos o el de almacenes key-value.5
  • Todo el código fuente necesario para la aplicación esté en un repositorio de git y éste esté hospedado en github. Asimismo, deberá incluir las instrucciones para ejecutarla en un ambiente de desarrollo en un archivo README.5

Por puntaje extra

Puede realizar cualquiera de los tres, por 5 puntos extra
  1. Permitir al usuario exportar una writeboard en formato html o texto plano
  2. Permitir al usuario invitar a colaboradores mediante correo electrónico.
  3. Agregar un mecanismo de control de concurrencia: si otro usuario ya está editando la writeboard, advertirle de ello a un nuevo visitante (pruebe esto en writeboard.com creando una y luego ingresando a ella con otro navegador web, debería ver un mensaje de advertencia).
